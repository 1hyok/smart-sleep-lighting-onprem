## 6.3 Free Tier 적용 시

학기 단일 사용자 트래픽은 대부분 **상시 무료(Always Free)** 한도 내:
- DynamoDB On-Demand: 월 25 WCU/RCU 상당 + 25GB 저장 무료(계정 기준) → 사실상 **$0**
- Lambda: 월 100만 호출 + 40만 GB-s 무료 → **$0**
- IoT Core: 신규 계정 12개월 250만 메시지/월 무료 → **$0** (12개월 이후 ~$0.86)
- API Gateway/CloudFront/S3: 프리티어 내 → ≈ **$0**

**결론**: 단일 사용자 기준 **월 실부담 ≈ $0**, Free Tier 만료/초과 시에도 **월 $5 미만**.
Aurora Serverless v2(최소 0.5 ACU ≈ 월 $40+) 대비 **8배 이상 저렴** → §4.2 DynamoDB 선택의 정량 근거.

## 6.4 민감도 — 규모 확장 시 (선형성 확인)

| 시나리오 | 변동 요인 | 월 비용(Free Tier 전, 근사) |
|---|---|---|
| 디바이스 10대 | IoT 메시지·DynamoDB 쓰기 ×10 | ~$34 (쓰기 $24 + IoT $8.6 지배) |
| 사용자 100명 | Secrets ×100, 폴러·리포트 ×100, API ×100 | ~$45 (Secrets $40 지배) |
| 발행 주기 3s→10s | 조도 트래픽 ×0.3 | ~$1.5 (쓰기·IoT 급감) |

**비용 동인 요약**: 디바이스 수↑ → 쓰기/IoT, 사용자 수↑ → **Secrets Manager($0.40/개)**.
다중 사용자 단계에선 Secrets 통합(사용자별 키 1 secret 다중 필드)이나 발행 주기 완화로 최적화 가능.

## 6.5 미달성/재평가 항목
- 본 추정은 가정 기반. 실배포 후 CloudWatch 실측으로 보정 필요.
- 트래픽이 안정·예측 가능해지면 DynamoDB **On-Demand → Provisioned** 전환 재평가(보고서 §4.5, §7).
- CloudWatch Logs 보존기간(기본 무기한) 단축으로 저장비 추가 절감 가능.
